Paradigms

Agent-oriented
Agent-oriented

Agent-oriented programming simulates the actions and interactions of autonomous agents.

Array Programming
Array Programming

In computer science, array programming refers to solutions that allow the application of operations to an entire set of values at once. Such solutions are commonly used in scientific and engineering settings.

Aspect-oriented
Aspect-oriented

Programming paradigm that aims to increase modularity by allowing the separation of cross-cutting concerns. This allows behaviors that are not central to the business logic (such as logging) to be added to a program without cluttering the code of core functions.

Assembly
Assembly

Assembly language is a low-level programming language for a computer, or other programmable device, in which there is a very strong (generally one-to-one) correspondence between the language and the architecture's machine code instructions.

Task-Driven Asynchronous Programming
Task-Driven Asynchronous Programming

Task-driven asynchronous programming allows tasks to be executed independently, enabling concurrency and efficient resource utilization.

Bidirectional Programming
Bidirectional Programming

Programs in which a single piece of code can be run in several ways, such that the same data are sometimes considered as input, and sometimes as output.

Compiled Language
Compiled Language

A compiled language is a programming language for which source code is typically compiled; not interpreted. The term is vague since, in principle, any language can be compiled or interpreted and in practice some languages are both (in different environments). In some environments, source code is first compiled (to an intermediate form; often bytecode) and then interpreted.

Concatenative Programming
Concatenative Programming

A concatenative programming language is a point-free computer programming language in which all expressions denote functions, and the juxtaposition of expressions denotes function composition.

Concurrent
Concurrent

Concurrent computing is a form of computing in which several computations are executed during overlapping time periods, concurrently, instead of sequentially. Concurrent computing is a form of modular programming. In its paradigm an overall computation is factored into subcomputations that may be executed concurrently.

Constraint Programming
Constraint Programming

In constraint programming, users declaratively state the constraints on the feasible solutions for a set of decision variables.

Design by Contract
Design by Contract

Design by Contract is a software design approach that uses contracts to define the obligations and guarantees of software components.

Data-centric Programming
Data-centric Programming

Programs describe the data to be matched and the processing required rather than defining a sequence of steps to be taken.

Data Exchange
Data Exchange

Data exchange languages allow data to be shared between systems in a structured format.

Data and Procedure Hiding
Data and Procedure Hiding

Data and procedure hiding restrict access to certain parts of a program's code.

Dataflow Programming
Dataflow Programming

Models a program as a directed graph of the data flowing between operations, thus implementing dataflow principles and architecture.

Declarative Programming
Declarative Programming

A style of building the structure and elements of computer programs—that expresses the logic of a computation without describing its control flow.

Distributed Programming
Distributed Programming

Distributed programming involves multiple computers working together on a single task.

Domain-Specific Language
Domain-Specific Language

A domain-specific language is tailored to a particular application domain.

Educational Programming
Educational Programming

Educational programming languages are designed mainly to teach programming concepts.

Embedded Systems
Embedded Systems

Embedded systems are specialized computing systems integrated into larger systems.

Esoteric Programming
Esoteric Programming

Esoteric programming languages are created as experiments, jokes, or for artistic purposes.

Event-Driven Programming
Event-Driven Programming

Event-driven programming centers around responding to user actions or other events.

Expression-Oriented Programming
Expression-Oriented Programming

Expression-oriented programming emphasizes the use of expressions over statements.

Expert System
Expert System

Expert systems use rule-based algorithms to simulate decision-making.

Extensible Programming
Extensible Programming

Extensible programming languages are designed to be extended with new features.

Functional Reactive Programming
Functional Reactive Programming

Functional reactive programming is a programming paradigm for reactive systems using functional programming.

Functional Programming
Functional Programming

A programming paradigm where programs are constructed by applying and composing functions. It is a declarative programming paradigm in which function definitions are trees of expressions that map values to other values.

General-Purpose Programming
General-Purpose Programming

General-purpose programming languages are designed to solve a wide range of problems.

Imperative Programming
Imperative Programming

Imperative programming uses statements to change a program's state.

Interpreted Programming
Interpreted Programming

Interpreted languages are executed directly, without prior compilation.

Language-Oriented Programming
Language-Oriented Programming

Language-oriented programming involves the creation of domain-specific languages to solve specific problems.

Lazy Evaluation
Lazy Evaluation

Lazy evaluation delays the evaluation of an expression until its value is needed.

Logic Programming
Logic Programming

Logic programming is based on formal logic, using facts and rules to derive conclusions.

Low-Code Development
Low-Code Development

Low-code platforms enable software development with minimal hand-coding.

Macro Processor
Macro Processor

A macro (short for "macro instruction") is a rule or pattern that specifies how a certain input should be mapped to a replacement output.

Message Passing
Message Passing

Message passing involves communication between processes through the exchange of messages.

Metaprogramming
Metaprogramming

Metaprogramming refers to writing programs that generate or manipulate other programs.

Modular Programming
Modular Programming

Modular programming involves dividing a program into separate, independent modules.

Multi-paradigm
Multi-paradigm

A programming paradigm is a relatively high-level way to conceptualize and structure the implementation of a computer program. Many/most languages are based on more than one paradigm.

Multiple Dispatch
Multiple Dispatch

Multiple dispatch selects a function to invoke based on the runtime types of multiple arguments.

Natural Language Programming
Natural Language Programming

Natural language programming enables the development of software using human language commands.

Object-oriented
Object-oriented

Programming paradigm based on the concept of objects, which can contain data and code: data in the form of fields (often known as attributes or properties), and code in the form of procedures (often known as methods). In OOP, computer programs are designed by making them out of objects that interact with one another.

Parallel Programming
Parallel Programming

Parallel programming executes multiple tasks simultaneously to improve performance.

Pattern-Matching
Pattern-Matching

Pattern-matching allows for checking and decomposing data structures based on patterns.

Pipeline Programming
Pipeline Programming

Pipeline programming passes data through a series of processing stages, each stage producing input for the next.

Point-free Programming
Point-free Programming

Point-free (aka tacit) programming avoids specifying arguments in function definitions, focusing on composition rather than application.

Procedural Programming
Procedural Programming

Procedural programming focuses on the use of procedures or functions to perform tasks.

Process-Oriented Programming
Process-Oriented Programming

Process-oriented programming is based on the concept of processes as the primary abstraction.

Protocol-Oriented Programming
Protocol-Oriented Programming

Protocol-oriented programming focuses on the design and use of interfaces or protocols to define program behavior.

Prototype-Based Programming
Prototype-Based Programming

Prototype-based programming is a style of object-oriented programming where objects are created by cloning existing ones.

Quantum Programming
Quantum Programming

Quantum programming deals with the development of algorithms for quantum computers.

Query Language
Query Language

A query language is used to make queries in databases and information systems.

Reactive Programming
Reactive Programming

Reactive programming focuses on asynchronous data streams and the propagation of change.

Real-Time Computing
Real-Time Computing

Real-time computing involves tasks that must be completed within strict timing constraints.

Reflective Programming
Reflective Programming

Reflective programming allows a program to inspect and modify its own structure and behavior at runtime.

Term Rewriting
Term Rewriting

Term-rewriting involves transforming expressions according to a set of rules to perform computation.

Rule-Based Processing
Rule-Based Processing

Rule-based processing involves defining rules that control the logic of decision-making systems.

Scientific Workflow
Scientific Workflow

Scientific workflow systems support the automation of complex scientific processes.

Scripting Language
Scripting Language

Scripting languages are used for automating tasks within a software environment.

Shared Memory
Shared Memory

Shared memory systems allow multiple processes to access the same memory space.

Simulation Language
Simulation Language

Simulation languages are specialized for modeling and simulating complex systems.

Stack-Oriented Programming
Stack-Oriented Programming

Stack-oriented programming uses a stack-based data structure for passing parameters.

Structured Programming
Structured Programming

Structured programming emphasizes control structures like loops and conditionals, avoiding the use of 'goto' statements.

Symbolic Programming
Symbolic Programming

Symbolic programming involves manipulating symbols rather than numeric data.

Synchronous Programming
Synchronous Programming

Synchronous programming operates in a time-driven manner where operations occur sequentially and in sync with a clock.

Data Validation
Data Validation

Data validation ensures that data conforms to predefined rules before it is processed.

Visual Programming
Visual Programming

A programming language that lets users create programs by manipulating program elements graphically rather than by specifying them textually.
