Type Systems

#
Affine Type System

An affine type system enforces that variables are used at most once, allowing for optimizations such as memory reuse.

#
Algebraic Data Types

In computer programming, especially functional programming and type theory, an algebraic data type (ADT) is a kind of composite type, i.e., a type formed by combining other types.

#
Dependent Typing

Dependent typing is a type system where types depend on terms, allowing for more expressive type constraints.

#
Duck Typing

Duck typing is a style of dynamic typing where an object's behavior determines its type rather than its inheritance.

#
Dynamic Typing

Dynamic typing defers type checking until runtime, allowing variables to hold different types at different times.

#
Flow-Sensitive Typing

Flow-sensitive typing is a type system where the types of variables can change based on control flow in the program.

#
Generic Programming

Generic programming uses algorithms that can work with several different but related data types.

#
Gradual Typing

Gradual typing allows mixing statically and dynamically typed code, progressively introducing type annotations.

#
Inferred Type System

A type inference system where types are automatically deduced without requiring explicit type annotations.

#
Latent Typing

Latent typing is a dynamic typing model where type constraints are deferred until runtime, similar to duck typing.

#
Manifest Typing

Manifest typing is a type system in which all variables are explicitly typed in the code, usually associated with static typing.

#
Nominal Typing

Nominal typing is a type system where compatibility between types is determined by explicit declarations or names, rather than structure.

#
Object-Oriented Typing

Object-oriented typing refers to type systems that manage and enforce types for objects and classes, focusing on inheritance and polymorphism.

#
Optional Typing

Optional typing allows for type annotations to be included or omitted, offering flexibility between static and dynamic typing.

#
Polymorphic Typing

Polymorphic typing allows types to be parameterized and reused with different data types, enhancing code reusability.

#
Run-Time Type Information (RTTI)

RTTI is a feature that allows a program to retrieve type information about an object during runtime.

#
Type Safe

Type safety ensures that operations are performed on compatible types, preventing unintended behavior and runtime errors.

#
Static Typing

Static typing checks type correctness at compile-time, reducing runtime errors by ensuring variables are properly typed before execution.

#
String Typing

String typing refers to the type system's handling and validation of string data types in programming languages.

#
Strong Typing

Strong typing enforces strict type rules, ensuring that variables cannot be implicitly cast or used in an unintended way.

#
Structural Typing

Structural typing bases type compatibility on the structure of data rather than explicit type declarations or names.

#
Uniqueness Typing

Uniqueness typing enforces that certain values are used only once, allowing optimizations like in-place updates in functional programming.

#
Untyped

Untyped systems lack explicit types, with variables and functions being dynamically interpreted based on their usage.

#
Weak Typing

Weak typing allows for more flexible type conversions, where variables can be implicitly cast between types, sometimes leading to unexpected results.
